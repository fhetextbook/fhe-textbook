D-5.9 Decomposing Multiplication: DecompMultRNS

In FHE, gadget decomposition (§A-6.4) is used to compute ciphertext-to-plaintext multiplication with a small noise. For example, BFV and CKKS’s homomorphic key switching (Summary C-5 in §C-5) uses gadget decomposition to compute 𝖱𝖫𝖶𝖤S′,σ(ΔM) = B + A ⋅𝖱𝖫𝖶𝖤S′,σ(S) with a small noise (where each coefficient of the polynomial A can be any value within the range of the ciphertext modulus q). As another example, the relinearization process of ciphertext-to-ciphertext multiplication in BFV (Summary D-2.7.5 in §D-2.7.5), CKKS (Summary D-3.5.6 in §D-3.5.6), and BGV (Summary D-4.8 in §D-4.8) uses gadget decomposition to derive the synthetic ciphertext 𝖱𝖫𝖶𝖤S′,σ(D2 ⋅S2) when computing 𝖱𝖫𝖶𝖤S′,σ(Δ2M⟨1⟩M⟨2⟩) = D0 + D1 ⋅S + D2 ⋅S2 = 𝖼𝗍α + 𝖼𝗍β, where 𝖼𝗍α = (D0,D1), 𝖼𝗍β = 𝖱𝖫𝖶𝖤S′,σ(D2 ⋅S2), D0 = B1B2, D1 = A1B2 + A2B1, and D2 = A1A2. Using gadget decomposition, we showed the following relations:

𝖱𝖫𝖶𝖤S′,σ(A ⋅S) = ⟨𝖣𝖾𝖼𝗈𝗆𝗉β,l(A), 𝖱𝖫𝖾𝗏S′,σβ,l(S)⟩ # used in key switching

𝖱𝖫𝖶𝖤S′,σ(D2 ⋅S2) = ⟨𝖣𝖾𝖼𝗈𝗆𝗉β,l(D2), 𝖱𝖫𝖾𝗏S,σβ,l(S2)⟩ # used in relinearization

However, if we convert a value (e.g., x) into an RNS vector, then it cannot be directly expressed in a gadget-decomposed form based on the β and l parameters. Instead, given the relationship between the value x and its RNS residues is x = ∑ ⁡ i=1kx ⋅q qi ⋅ |(q qi−1)| qi mod q, we can treat each RNS residue as a gadget-decomposed element. For example, suppose our goal is to decompose 𝖱𝖫𝖶𝖤S′,σ(A ⋅S), where the RNS vector of A = (A1,A2,⋯,Ak) and whose base moduli are (q1,q2,⋯,qk). We can decompose 𝖱𝖫𝖶𝖤S′,σ(A ⋅S) as follows:

𝖱𝖫𝖶𝖤S′,σ(A ⋅S) mod q

= 𝖱𝖫𝖶𝖤S′,σ (S ⋅ (A1 q q1 ⋅ |( q q1) −1| q1 + A2 q q2 ⋅ |( q q2) −1| q2 + ⋯ + Ak q qk ⋅ |( q qk) −1| qk)) mod q

= 𝖱𝖫𝖶𝖤S′,σ (S ⋅A1 q q1 ⋅ |( q q1) −1| q1) + 𝖱𝖫𝖶𝖤S′,σ (S ⋅A2 q q2 ⋅ |( q q2) −1| q2) +

⋯ + 𝖱𝖫𝖶𝖤S′,σ (S ⋅Ak q qk ⋅ |( q qk) −1| qk) mod q

= A1 ⋅𝖱𝖫𝖶𝖤S′,σ (S ⋅ q q1 ⋅ |( q q1) −1| q1) + A2 ⋅𝖱𝖫𝖶𝖤S′,σ (S ⋅ q q2 ⋅ |( q q2) −1| q2) +

⋯ + Ak ⋅𝖱𝖫𝖶𝖤S′,σ (S ⋅ q qk ⋅ |( q qk) −1| qk) mod q

= ∑ ⁡ i=1k (Ai ⋅𝖱𝖫𝖶𝖤S′,σ (S ⋅q qi ⋅ |(q qi) −1| qi)) mod q

, where {𝖱𝖫𝖶𝖤S′,σ (S ⋅q qi ⋅ |(q qi) −1| qi)} i=1k can be pre-generated as RNS key-switching keys.

Applying the same reasoning as the above, we can also derive the following for relinearization:

𝖱𝖫𝖶𝖤S,σ(D2 ⋅S2) = ∑ ⁡ i=1k(D2,i ⋅𝖱𝖫𝖶𝖤S,σ (S2 ⋅q qi ⋅ |(q qi) −1| qi) ) mod q

, where {𝖱𝖫𝖶𝖤S,σ (S2 ⋅q qi ⋅ |(q qi) −1| qi)} i=1k can be pre-generated as relinearization keys.

RNS-based multiplication decomposition is summarized as follows:

⟨Summary D-5.9⟩ DecompMultRNS

For key-switching:

Input: A = (A1,A2,⋯,Ak) ∈R⟨n,q1⟩×R⟨n,q2⟩×⋯ ×R⟨n,qk⟩,

S⟨S′,𝖱𝖭𝖲⟩ = {𝖱𝖫𝖶𝖤S′,σ (S ⋅q qi ⋅ |(q qi) −1| qi)} i=1k # key-switching keys

𝖣𝖾𝖼𝗈𝗆𝗉𝖬𝗎𝗅𝗍𝖱𝖭𝖲(A,S⟨S′,𝖱𝖭𝖲⟩) = 𝖱𝖫𝖶𝖤S′,σ(A ⋅S)

= ∑ ⁡ i=1k(Ai ⋅𝖱𝖫𝖶𝖤S′,σ (S ⋅q qi ⋅ |(q qi) −1| qi) ) mod q

For relinearization:

Input: D2 = (D2,1,D2,2,⋯,D2,k) ∈R⟨n,q1⟩×R⟨n,q2⟩×⋯ ×R⟨n,qk⟩,

S⟨S,𝖱𝖭𝖲⟩2 = {𝖱𝖫𝖶𝖤S,σ (S2 ⋅q qi ⋅ |(q qi) −1| qi)} i=1k # relinearization keys

𝖣𝖾𝖼𝗈𝗆𝗉𝖬𝗎𝗅𝗍𝖱𝖭𝖲(D2,S⟨S,𝖱𝖭𝖲⟩2) = 𝖱𝖫𝖶𝖤S,σ(D2 ⋅S2)

= ∑ ⁡ i=1k(D2,i ⋅𝖱𝖫𝖶𝖤S,σ (S2 ⋅q qi ⋅ |(q qi) −1| qi) ) mod q
