C-3 GLWE Ciphertext-to-Plaintext Multiplication

- Reference: TFHE Deep Dive - Part II - Encodings and linear leveled operations [8]

Suppose we have a GLWE ciphertext C:

C = 𝖦𝖫𝖶𝖤S,σ(M) = (A1,A2,... Ak−1,B) ∈R⟨n,q⟩k+1

and a new plaintext polynomial Λ as follows:

Λ = ∑ ⁡ i=0n−1(Λi ⋅Xi) ∈R⟨n,q⟩

Let’s define the following ciphertext-to-plaintext multiplication operation:

Λ ⋅C = (Λ ⋅A0,Λ ⋅A1,... Λ ⋅Ak−1,Λ ⋅B)

We assume that we always do polynomial-to-polynomial multiplications efficiently in O(nlog⁡n) by using the NTT technique (§A-16). Then, the following is true:

⟨Summary C-3⟩ GLWE Ciphertext-to-Plaintext Multiplication

Λ ⋅𝖦𝖫𝖶𝖤S,σ(ΔM)

= Λ ⋅({Ai⟨1⟩}i=0k−1, B⟨1⟩)

= ({Λ ⋅Ai⟨1⟩}i=0k−1, Λ ⋅B⟨1⟩)

= 𝖦𝖫𝖶𝖤S,σ(Δ(M ⋅Λ))

This means that multiplying a plaintext (polynomial) by a ciphertext (polynomial) and decrypting it gives the same result as multiplying the two original plaintext polynomials.

Proof

1.
Define the following notations:
A1′ = Λ ⋅A1
A2′ = Λ ⋅A2
...
Ak−1′ = Λ ⋅Ak−1
E′ = Λ ⋅E
B′ = Λ ⋅B
2.
Derive the following:
B′ = Λ ⋅B
= Λ ⋅(∑ ⁡ i=0k−1(Ai ⋅Si) + Δ ⋅M + E) = ∑ ⁡ i=0k−1(Λ ⋅Ai ⋅Si) + Δ ⋅Λ ⋅M + Λ ⋅E
# by the distributive property of a polynomial ring
= ∑ ⁡ i=0k−1((Λ ⋅Ai) ⋅Si) + Δ ⋅(Λ ⋅M) + (Λ ⋅E)
= ∑ ⁡ i=0k−1(Ai′⋅Si) + Δ ⋅(Λ ⋅M) + (E′)
3.
Since B′ = ∑ ⁡ i=0k−1(Ai′⋅Si) + Δ ⋅(Λ ⋅M) + (E′),

(A1′,A2′,... Ak−1′,B′) form the ciphertext 𝖦𝖫𝖶𝖤S,σ(Δ ⋅Λ ⋅M).

4.
Thus,
Λ ⋅𝖦𝖫𝖶𝖤S,σ(ΔM)
= (Λ ⋅A0, Λ ⋅A1,... Λ ⋅Ak−1, Λ ⋅B)
= ({Ai′}i=0k−1, Λ ⋅B)
= 𝖦𝖫𝖶𝖤S,σ(Δ(M ⋅Λ))

□

If we decrypt 𝖦𝖫𝖶𝖤S,σ(Δ ⋅Λ ⋅M) by using S, then we get the plaintext Λ ⋅M. Meanwhile, A1′,A2′,... Ak−1′,E′ get eliminated by rounding during decryption, regardless of whatever their values were randomly sampled during encryption.

The noise is a bigger problem now, because after decryption, the original ciphertext C’s noise has increased from E to E′ = Λ ⋅E. This means that if we continue multiplication computations without decrypting the ciphertext to blow away the noise E′, it will continue growing more and eventually the noise in the lower bit area in B will overflow to the scaled plaintext bit area. If this happens, the noise E′ won’t be blown away during decryption, ending up corrupting the plaintext M. Therefore, if the constant Λ is big, it is recommended to use gadget decomposition (§A-6.4), which we will explain in the next subsection.

C-3.0.1 Gadget Decomposition for Noise Suppression
C-3.0.1 Gadget Decomposition for Noise Suppression

In ciphertext-to-plaintext multiplication Λ ⋅𝖦𝖫𝖶𝖤S,σ(M), the noise E grows to E′ = Λ ⋅E. To limit this noise growth, we introduce a technique based on decomposing Λ (§A-6.1) and an GLev encryption (§B-5.1) of M as follows:

Λ = Λ1 q β1 + Λ2 q β2 + ⋯ + Λl q βl→𝖣𝖾𝖼𝗈𝗆𝗉β,l(Λ) = (Λ1,Λ2,⋯,Λl)

𝖦𝖫𝖾𝗏S,σβ,l(ΔM) = {𝖦𝖫𝖶𝖤S,σ (ΔM q β1) ,𝖦𝖫𝖶𝖤S,σ (ΔM q β2) ,⋯𝖦𝖫𝖶𝖤S,σ (ΔM q βl) }

We will encrypt the plaintext M as 𝖦𝖫𝖾𝗏S,σβ,l(ΔM) instead of 𝖦𝖫𝖶𝖤S,σ(ΔM), and compute 𝖣𝖾𝖼𝗈𝗆𝗉(𝖣𝖾𝖼𝗈𝗆𝗉β,l(Λ) ⋅𝖦𝖫𝖾𝗏S,σβ,l(ΔM) instead of Λ ⋅𝖦𝖫𝖶𝖤S,σ(ΔM). Notice that the results of both computations are the same as follows:

𝖣𝖾𝖼𝗈𝗆𝗉β,l(Λ) ⋅𝖦𝖫𝖾𝗏S,σβ,l(ΔM)

= (Λ1,Λ2,⋯,Λl) ⋅ (𝖦𝖫𝖶𝖤S,σ ( q βΔM ), 𝖦𝖫𝖶𝖤S,σ ( q β2ΔM ), ⋯, 𝖦𝖫𝖶𝖤S,σ ( q βlΔM ))

= Λ1 ⋅𝖦𝖫𝖶𝖤S,σ ( q βΔM ) + Λ2 ⋅𝖦𝖫𝖶𝖤S,σ ( q β2ΔM ) + ⋯ + Λl ⋅𝖦𝖫𝖶𝖤S,σ ( q βlΔM )

= 𝖦𝖫𝖶𝖤S,σ (Λ1 ⋅q βM ) + 𝖦𝖫𝖶𝖤S,σ (Λ2 ⋅ q β2M ) + ⋯ + 𝖦𝖫𝖶𝖤S,σ (Λl ⋅ q βlM )

= 𝖦𝖫𝖶𝖤S,σ (Λ1 ⋅q βΔM + Λ2 ⋅ q β2ΔM + ⋯ + Λl ⋅ q βlΔM )

= 𝖦𝖫𝖶𝖤S,σ ((Λ1 ⋅q β + Λ2 ⋅ q β2 + ⋯ + Λl ⋅ q βl) ⋅ΔM )

= 𝖦𝖫𝖶𝖤S,σ (Λ ⋅ΔM )

= Λ ⋅𝖦𝖫𝖶𝖤S,σ(ΔM)

While the computation results are the same, as we decompose Λ into smaller plaintext polynomials Λ1,Λ2,⋯,Λl, the generated noise by each of l plaintext-to-ciphertext multiplications becomes smaller. Given the noise of each GLWE ciphertext in the GLev ciphertext is Ei, the final noise of the ciphertext-to-plaintext multiplication is ∑ ⁡ i=0lEi, which is much smaller than Λ ⋅E, because l is much smaller than Λ.

However, we cannot use this decomposition technique to the resulting ciphertext again, because the output of this algorithm is a GLWE ciphertext and converting it into a GLev ciphertext without decrypting it costs much noise and computation time (as we need to multiply the GLWE ciphertext by q β, q β2,⋯, q βl).

As a more efficient technique to re-initialize the noise E, we will describe TFHE’s noise bootstrapping technique in §D-1.8.
